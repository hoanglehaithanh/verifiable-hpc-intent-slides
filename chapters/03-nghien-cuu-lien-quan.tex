\begin{frame}[t]{3.1. LLM và alternative resource profiles}
\footnotesize
\begin{columns}[T,onlytextwidth]
\begin{column}{0.49\textwidth}
\begin{block}{LLM trong scheduling}
\begin{itemize}
\item phân tích constraint từ mô tả tự nhiên;
\item cân bằng nhiều mục tiêu;
\item sinh structured output;
\item cung cấp diễn giải bằng ngôn ngữ tự nhiên.
\end{itemize}
\end{block}
\begin{alertblock}{Rủi ro của raw LLM output}
\begin{itemize}
\item sai constraint hoặc arithmetic;
\item commit quá sớm;
\item không tách hard constraint và preference.
\end{itemize}
\end{alertblock}
\end{column}
\begin{column}{0.49\textwidth}
\begin{block}{Moldable và heterogeneous scheduling}
\begin{itemize}
\item waiting time;
\item turnaround;
\item utilization;
\item khả năng khai thác tài nguyên không đồng nhất.
\end{itemize}
\end{block}
\begin{alertblock}{Giả định còn hạn chế}
\begin{itemize}
\item alternative allocations đã có sẵn;
\item performance model đã biết;
\item application hỗ trợ scaling;
\item user flexibility đã được biểu diễn rõ.
\end{itemize}
\end{alertblock}
\end{column}
\end{columns}
\end{frame}

\begin{frame}[t,shrink=20]{3.3. Các nghiên cứu về prediction và uncertainty}
\small
Runtime, memory và GPU usage prediction là các thành phần quan trọng trong HPC resource management.\par
Các hướng nghiên cứu hiện có thường tập trung vào:\par
\begin{itemize}
\item dự đoán runtime;
\item dự đoán memory;
\item dự đoán utilization;
\item sử dụng telemetry hoặc historical logs;
\item xây dựng prediction interval.
\end{itemize}
Tuy nhiên, prediction thường được xem là output hoặc feature đầu vào cho scheduler, chưa luôn được ánh xạ thành quyết định commit, keep original hoặc schedule probe.\par
\end{frame}

% Source slide 21
\begin{frame}[t,shrink=20,fragile]{3.4. Các nghiên cứu về queue-aware decision}
\small
Queue state có ảnh hưởng trực tiếp đến turnaround time của user.\par
Các scheduler truyền thống như backfilling đã khai thác queue state để cải thiện hiệu quả hệ thống.\par
Tuy nhiên, queue-aware scheduling thường hoạt động ở tầng scheduler, trong khi bài toán của chuyên đề nằm ở tầng trước submission:\par
\begin{block}{Minh họa}
\begin{verbatim}
User request
      ↓
Profile recommendation
      ↓
Slurm scheduler không thay đổi
\end{verbatim}
\end{block}
Do đó, cần tách rõ profile recommendation trước submission và scheduling decision bên trong scheduler.\par
\end{frame}

% Source slide 22
\begin{frame}[t,shrink=20]{3.5. Đánh giá chung}
\small
Các nghiên cứu hiện có đã giải quyết nhiều thành phần quan trọng:\par
\begin{itemize}
\item LLM có thể hỗ trợ hiểu constraint và sinh structured output.
\item Moldable scheduling cho thấy lợi ích của alternative allocation.
\item Prediction model có thể ước lượng runtime và resource usage.
\item Queue-aware scheduling cho thấy trạng thái hệ thống ảnh hưởng lớn đến turnaround.
\end{itemize}
Tuy nhiên, còn thiếu một pipeline được đánh giá end-to-end để chuyển:\par
\[
\text{Slurm request + natural-language intent + queue state + limited evidence}
\]
thành:\par
\[
\text{resource-profile decision có uncertainty, evidence và khả năng abstain}
\]
trước khi submit job.\par
\end{frame}
